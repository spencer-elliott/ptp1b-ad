Analyzing the Feature-Barcode Matrix in R

The main output from pre-processing is a matrix quantifying the expression of each gene in each cell. The Seurat package for R includes many tools for analyzing this type of single-cell data.

This analysis will heavily follow the Guided Clustering Tutorial provided on the Seurat website: https://satijalab.org/seurat/articles/pbmc3k_tutorial.html

Creating a Seurat object from the matrix

Let’s start by loading in the Seurat package. Since the data is in matrix format, I will also use the Matrix package for some easier manipulation. Lastly the dplyr package is always helpful for piping commands and moving data around.

dim(counts)
[1] 372063  55335

That represents 372,063 possible cell barcodes, and 55,335 genes/transcripts from the reference transcriptome (a high number because this includes many noncoding).

First of all this matrix needs to be transposed, Seurat expects rows to be features and columns to be cell barcodes:

counts <- t(counts)

Next I will bring in the lists of barcode and feature names:

barcodes <- readLines("counts.barcodes.txt")
features <- readLines("counts.genes.txt")

The feature names are currently Ensembl IDs that need the version number removed.

features <- sub("\\.\\d+$", "", features)

These lists can be assigned as names to the columns and rows of the matrix:

rownames(counts) <- features
colnames(counts) <- barcodes

372k barcodes is an extremely high amount, it’s likely that many of these are empty droplets that don’t actually contain cells. Printing a summary of UMIs supports this conclusion:

summary(colSums(counts))
   Min. 1st Qu.  Median    Mean 3rd Qu.    Max. 
    0.0     1.0     1.0    65.4    25.0 34963.0 
# colSums(): Adds up the values in each column (# of UMIs for each cell)
# summary(): Computes some basic statistics (mean, median, min, max)

The mean is 65.4 while the median is only 1! That means at least half the barcodes only have 1 UMI, a sign of empty droplets. To get rid of these data points, I will filter down to barcodes associated with at least 500 UMIs.

counts_filtered <- counts[, colSums(counts) >= 500]

Now the matrix with raw counts can be turned into Seurat’s special data structure, which can store the analysis that I will perform. This step also allows for a bit of additional filtering:

so <- CreateSeuratObject(counts = counts_filtered,

I chose these numbers for filtering as a starting point (they are used in the walkthrough example on Seurat’s website). There is also more quality control to look at before drawing any big conclusions.

Filtering Low-Quality Cells

We need to remove low-quality cells from this scRNA-seq data. There are a few common ways to filter out low-quality and dying cells, including:

- Low number of total UMIs.
- Low number of unique genes/features.
- High percentage of UMIs from mitochondrial genes. (Cytoplasmic RNA is lost first in dying cells, so mitochondria get overrepresented)

The first two are built into the cell metadata that Seurat gives us as “nCount_RNA” and “nFeature_RNA,” respectively:

Based on these plots we can filter out the outliers and make this data better for analysis. The Seurat tutorial example uses the following numbers:

  • 200 < nFeature_RNA < 2500

  • percent.mt < 5

That data is made up of human cells with different results for the previous figures. Looking at my own plots, it looks like I can be a little more generous and include:

  • 200 < nFeature_RNA < 5000

  • percent.mt < 15

This will keep the main body of each plot while removing almost all doublets and dead cells. Time to apply these filters to the data:

so <- subset(so, subset = nFeature_RNA > 200 &
                           nFeature_RNA < 5000 &
                           percent.mt < 15)

Normalization

Right now the data is still raw counts, so a cell with more total UMIs will look like it has more expression for each gene. Data is commonly normalized so gene expression is given in proportion to total UMIs:

so <- NormalizeData(so, normalization.method = "LogNormalize", scale.factor = 10000)

Scaling / PCA

The next goal is to do a PCA, which condenses all of these variable genes into a few principal components that clearly separate clusters of correlated cells. These separations should ideally reflect underlying biology, like the different cell types present with different levels of gene expression.

Similar to the normalization performed earlier, the data has to be scaled before proceeding as some genes are naturally expressed much more highly than others. For example, a widely used housekeeping gene will show much more expression across all cells versus a rarely used transcription factor. Scaling ensures that the principal components are not dominated by genes with this higher magnitude of expression.

all.genes <- rownames(so)
# rownames(so): List of all genes in the Seurat object

so <- ScaleData(so, features = all.genes)

Assigning Identity to Clusters

The CellMarker website allows you to look up marker genes: http://www.bio-bigdata.center/CellMarkerBrowse.jsp

A good starting point for identifying the clusters is automatic annotation. These programs can compare the data from these cells to a reference dataset with known labels. The result can give a great head start on identifying clusters that can then be manually checked.

For this analysis I’m going to try automatic annotation with SingleR. This requires a new library and the celldex library containing reference datasets:

SingleR uses the raw UMI counts from the Seurat object. The most important thing it generates is a vector of cell type predictions:

counts.singleR <- GetAssayData(so, layer = "counts")
# GetAssayData(): Extract a layer of data from the Seurat object
# layer = "counts": Raw UMI counts

pred <- SingleR(test = counts.singleR, ref = ref, labels = ref$label.main)
# test = counts.singleR: Data to annotate
# ref = ref: Reference dataset
# labels = ref$label.main: Chooses column from the reference to use as labels. This can be 'main' or 'fine.'

head(pred$labels, 10)
 [1] "Oligodendrocytes" "Oligodendrocytes" "Oligodendrocytes" "Oligodendrocytes"
 [5] "T cells"          "Epithelial cells" "Oligodendrocytes" "Oligodendrocytes"
 [9] "Oligodendrocytes" "Astrocytes"

Let’s add these labels to the Seurat object, and then use them to plot again:

so$SingleR.labels <- pred$labels

DimPlot(so, group.by = "SingleR.labels", reduction = "umap", label = TRUE)

Now the clusters are labeled with cell type predictions. I can start checking out the expression of one gene in different cell types, like this:

FeaturePlot(so, features = "Ptpn1")

This is the first step to verifying these cell types! For example, cluster 12 was predicted to be neurons by SingleR. Looking at a canonical marker gene for neurons, Snap25:

FeaturePlot(so, features = "Snap25")

It is most expressed in cluster 12! This is the type of analysis that will help verify these results. These canonical marker genes can come from knowledge/experience and can also be found in many online resources.
